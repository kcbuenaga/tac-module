# Thesis Timeline Planner - Workflow Flow

## CREATE MODE

```
┌─────────────────────────────────────────────────────────────┐
│            CREATE NEW THESIS TIMELINE                       │
│        Backward-Planning from Defense Deadline              │
└─────────────────────────────────────────────────────────────┘

Step 01: Init & Defense Date
  ├─ Welcome, explain backward-planning
  ├─ Collect defense deadline date
  ├─ Create timeline document from template
  └─ [C] Continue
      ↓
Step 02: Calculate Timeline
  ├─ Calculate days remaining
  ├─ Explain planning phases
  └─ [C] Continue
      ↓
Step 03: Chapter Milestones
  ├─ Discuss chapter structure
  ├─ Calculate chapter completion dates
  ├─ Append to timeline
  └─ [C] Continue
      ↓
Step 04: Weekly Goals
  ├─ Break down into weekly milestones
  ├─ Create actionable goals
  ├─ Append to timeline
  └─ [C] Continue
      ↓
Step 05: Finalize Timeline
  ├─ Review complete timeline
  ├─ Add progress tracking
  ├─ Present final timeline
  └─ END
```

## EDIT MODE

```
┌─────────────────────────────────────────────────────────────┐
│              EDIT EXISTING TIMELINE                         │
│           Update Milestones & Progress                      │
└─────────────────────────────────────────────────────────────┘

Step E-01: Assess Timeline
  ├─ Load existing timeline
  ├─ Show current milestones
  ├─ Ask what to edit
  └─ Route to edit step
      ├─ [1] Edit Defense Date → Step E-02
      ├─ [2] Edit Chapters → Step E-03
      ├─ [3] Edit Weekly Goals → Step E-04
      └─ [4] Update Progress → Step E-05

Step E-02/03/04/05: Make Edits
  ├─ Update selected section
  ├─ Recalculate dependent milestones
  ├─ Save changes
  └─ END
```

## VALIDATE MODE

```
┌─────────────────────────────────────────────────────────────┐
│            VALIDATE TIMELINE QUALITY                        │
│          Check Completeness & Feasibility                   │
└─────────────────────────────────────────────────────────────┘

Step V-01: Validate Timeline
  ├─ Load timeline
  ├─ Check all required sections
  ├─ Validate date logic
  ├─ Assess feasibility
  └─ Generate validation report
      ↓
Step V-02: Present Results
  ├─ Display findings
  ├─ Recommend improvements
  └─ END
```

## Key Features

- **Backward-Planning:** Works from defense deadline backwards
- **Tri-Modal:** Create, Edit, Validate lifecycle
- **Collaborative:** AI guides through realistic milestone setting
- **Anti-Procrastination:** Transforms distant deadline into weekly actions
